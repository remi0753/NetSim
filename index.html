スパイン・リーフ ファブリック生成

L3-to-the-leaf 構成を自動生成します。リンクごとに /30 サブネット、 OSPF (ECMP) で全経路を自動学習。ホストはラックごとにグループ化されます。

NetSim — L1〜L4 ネットワークシミュレーター

構築の流れ

  1. 左のパレットからデバイスを選び、キャンバスをクリックして配置
  2. ツールバーの「🔌 結線」でデバイス→ポートを2箇所選んでケーブル接続
  3. デバイスをクリック → 右パネルでIP/VLANなどを設定
    (または「コンソール」からCLIで設定)
  4. PCのコンソールから ping / traceroute / http get を実行し、パケットの流れを観察

マウス操作

  • ドラッグ: デバイス移動 / 空白部で画面パン
  • ホイール: ズーム
  • ダブルクリック: コンソールを開く
  • Delete: 選択中のデバイス/リンクを削除

PC / サーバのコマンド

  • set ip 10.0.1.11 255.255.255.0 10.0.1.254
  • ping 10.0.2.10 [-c 8]
  • traceroute 10.0.2.10
  • arp -a / ipconfig
  • http get 10.0.2.10 (TCPハンドシェイク観察)
  • udp send 10.0.2.10 5000 hello

スイッチ/ルータのCLI (Cisco IOS風)

  • enable → configure terminal
  • interface Gi0/1 → switchport access vlan 10
  • interface vlan 10 → ip address … (L3スイッチ)
  • ip route 0.0.0.0 0.0.0.0 10.0.0.1
  • router ospf 1 → network 10.0.0.0 0.255.255.255 area 0
  • vrrp 1 ip 10.0.1.1 / channel-group 1 mode active
  • ip helper-address 10.0.2.9 (DHCPリレー)
  • access-list 100 deny icmp any host 10.0.2.10
  • show ip route / show ip ospf neighbor / “?”でヘルプ
  • 省略形OK: sh ip int br, conf t

大規模構成

  • 「🏭 DC生成」: スパイン・リーフを自動生成 (OSPF+ECMP設定済み)
  • Shift+ドラッグ: 複数選択 → 一括IP設定/CLI一括実行/グループ化
  • グループ(ラック)はダブルクリックで展開/インスペクタで折りたたみ
