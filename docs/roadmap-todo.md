# NetSim 拡張 TODO — インターネット規模と段階学習の両立

## この TODO の前提

現状は、L1〜L4 のパケットを 1 フレームずつ追跡でき、スパイン 4・リーフ 8・各リーフ
48 ホスト（最大 396 ホスト）のファブリックを生成できる。OSPF/ECMP を含む 200 ホストの
性能テストもある。一方、リンク遅延は全リンク一律、OSPF は単一エリアの簡略実装で、
STP・IPv6・BGP は未実装である。

「インターネット規模」を単に端末を大量配置することと定義しない。次の二つを両立させる。

- **詳細モード**: 数台〜数千ノードで、L1〜L4 のフレーム・設定・障害を正確に観察する。
- **広域モード**: AS、プレフィックス、リンク容量を集約して、数万〜十万 AS 級の経路制御と障害波及を扱う。個々のフレームを全ホップで複製しない。

広域モードから任意の AS／PoP を詳細モードへ展開する「段階的詳細化」を共通の操作にする。

## 完了の目標と受け入れ基準

- [ ] **規模**: 広域モードで 10,000 AS・100,000 プレフィックスの収束、経路検索、単一リンク障害を開発用標準機で操作可能な時間内に完了する。具体的な CPU・メモリ・時間の予算は P0-1 のベンチマークで先に固定する。
- [ ] **観察性**: ユーザーが「この経路が選ばれた理由」と「障害後にどこで変わったか」を AS-PATH、ポリシー、時系列イベントから確認できる。
- [ ] **学習性**: Lv.0 から順に、操作説明なしで開始でき、各課題で自動判定・ヒント・復帰手段を利用できる。
- [ ] **互換性**: 既存の JSON 保存データ、サンプル、コア／ブラウザテストを壊さない。保存形式にはバージョン移行を設ける。

## P0 — 規模へ進むための基盤（最優先）

### P0-1: 性能の測定と上限の見える化

- [ ] `tests/benchmark.js` を追加し、固定シードで 200 / 1,000 / 10,000 ノードの生成、収束、障害、トラフィックを計測する。
- [ ] イベント数、ピークヒープ長、LSDB／RIB サイズ、描画時間、メモリ使用量を記録する。
- [ ] CI 用の小規模性能予算と、手元で実行する大規模ベンチマークを分離する。
- [ ] 起動時と生成ダイアログに「詳細モードで安全に扱える推定規模」と、制限に達した理由を表示する。

**完了条件**: 同じシードで結果を比較でき、最遅処理とボトルネックが数値で分かる。

### P0-2: シミュレーションを詳細／集約の二層にする

- [ ] `SimulationProfile`（詳細・集約・ハイブリッド）を定義し、トポロジーと保存形式に持たせる。
- [ ] 広域モードでは端末・L2・各パケットの代わりに、AS、PoP、集約リンク、プレフィックス、フロー統計をイベントとして扱う。
- [ ] AS／PoP を展開すると、その範囲だけ既存の詳細な L1〜L4 モデルに切り替える。
- [ ] 詳細範囲と集約範囲の境界で、到達性、遅延、損失、帯域、NAT／サービスの要約を受け渡すアダプターを実装する。
- [ ] `structuredClone` による全フレーム複製と無制限のイベント登録をプロファイルし、集約モードでは不必要な複製・アニメーションを発生させない。

**完了条件**: 同一の AS 間通信について、集約経路から選択した PoP を展開して、経路の対応関係を確認できる。

### P0-3: 大規模トポロジー入出力・生成

- [ ] DC 生成の固定上限（spine 4 / leaf 8 / host 48）を UI 制約ではなくプロファイル別の上限へ置換する。
- [ ] Clos、階層キャンパス、ISP、IXP、マルチリージョン、マルチ AS をテンプレート生成できるようにする。
- [ ] JSON にスキーマバージョン、検証、マイグレーション、部分読み込みを追加する。
- [ ] CSV/JSON から AS、プレフィックス、リンク、需要を取り込むインポーターを追加する。実データを読み込む場合の匿名化・ライセンス表示も行う。
- [ ] 固定シード、名前付け規則、IP／ASN の自動割当、重複検出を追加する。

**完了条件**: 生成した 10,000 AS 構成を保存・再読込でき、壊れた入力は原因付きで拒否される。

### P0-4: 大規模 UI と観測の間引き

- [ ] 画面外・折りたたみ済み要素を描画しない仮想化、ズーム段階別のクラスタ表示、検索・レイヤ／AS フィルターを実装する。
- [ ] パケットログの固定 500 行上限を、リングバッファ、プロトコル／AS／フロー／時間フィルター、サンプリング、統計集約に置き換える。
- [ ] 「全パケット可視化」と「統計のみ」を切り替え、規模に応じて安全な既定値を選ぶ。
- [ ] 経路差分、収束進行、ドロップ理由、キュー長、リンク使用率を時系列で表示する。

**完了条件**: 数千要素を開いてもパン・ズーム・検索が操作不能にならず、重要な通信だけを絞り込める。

## P1 — インターネットらしいネットワーク機能

### P1-1: BGP を最小核から段階実装する

- [ ] eBGP、ASN、CIDR プレフィックス広告／撤回、AS-PATH、NEXT_HOP、ループ防止を実装する。
- [ ] Best Path の説明可能な選択順序（LOCAL_PREF、AS-PATH 長、MED、eBGP/iBGP、IGP コスト）と `show bgp` を追加する。
- [ ] import/export ポリシー、prefix-list、route-map、コミュニティ、デフォルト経路を追加する。
- [ ] iBGP、route reflector、集約、max-prefix、graceful restart を追加する。
- [ ] RPKI/ROA の妥当性、経路リーク・ハイジャックのシナリオと可視化を追加する。

**完了条件**: 3 AS の基本広告から、ポリシーで経路を変え、障害・リーク・RPKI 拒否まで再現できる。

### P1-2: IP と内部ルーティングの拡張

- [ ] IPv6（アドレス、NDP、ICMPv6、デュアルスタック、RA/SLAAC、DHCPv6）を実装する。
- [ ] OSPF をエリア、ABR、LSA 種別、認証、タイマー、コスト／帯域連動へ段階拡張する。現在の単一エリア簡略実装は「学習モード」として残す。
- [ ] IS-IS を追加し、ISP テンプレートで OSPF と比較可能にする。
- [ ] VRF、MP-BGP の最小モデル、IPv4/IPv6 VPN を後続項目として分離し、実装順を BGP 安定化後にする。

### P1-3: L2、物理、トラフィックの現実性

- [ ] RSTP/MSTP（root bridge、ポートロール、収束）を実装し、LACP と役割の違いを可視化する。
- [ ] リンク単位の遅延、ジッター、帯域、MTU、損失率、エラー、キューと輻輳を設定できるようにする。
- [ ] DHCP のリース更新／期限、DNS（権威・再帰・キャッシュ）、NTP、基本的な HTTP/DNS クライアントを追加する。
- [ ] TCP の再送、RTT、ウィンドウ、輻輳制御を「簡易」と「観察可能なモデル」の二段階で追加する。
- [ ] QoS（分類、優先キュー、policing/shaping）とネットワーク障害注入（切断、劣化、誤設定）を追加する。

**完了条件**: 帯域飽和・MTU 不一致・STP 収束・DNS 障害を、パケットと統計の両方で説明できる。

## 推奨する実装順

1. P0-1（計測） → P0-4（描画・観測の間引き） → P0-2（集約モデル） → P0-3（大規模生成）。まず規模を測定可能かつ操作可能にする。
2. P1-1 の eBGP 最小核と 3 AS レッスンを同時に実装する。機能だけでなく、選択理由の説明 UI を必須とする。
3. IPv6、リンク特性、RSTP、DNS/TCP を、各々の導入レッスンと異常系テストを添えて追加する。
4. 共有・教材制作・外部形式は、シミュレーションとレッスンの保存形式が安定してから着手する。

## 今回は着手しないもの

- 実インターネットへのパケット送出や実ルータの設定変更。教育用シミュレーターの範囲外であり、安全性・権限・再現性を損なう。
- 「全世界の全ホスト」をフレーム単位で忠実に再現すること。広域モードでは AS／プレフィックス／フロー統計へ集約し、必要箇所だけ詳細化する。
